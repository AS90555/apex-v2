# Brief: P-01 — Optuna TPE statt Grid+MC im Auto-Lab

**Datum:** 2026-05-07
**Priorität:** P3
**Adressat:** lab-tuner

## Kontext
Auto-Lab nutzt Grid+MC über BATCH_SIZE=20 Kombinationen pro
(strategy, asset)-Runde. Mit MIN_PF_TEST=1.80, n≥100, apply_costs=True
und BH q=0.05 bestehen wenige Kombinationen — Grid+MC sucht blind.
Optuna TPE lernt aus vorherigen Trials und schlägt bessere Parameter vor.
Erwartete Verbesserung: 5–10× sample-effizienter bei gleicher Hit-Rate.

## Aufgabe

### Schritt 1 — Installation
pip install optuna --break-system-packages
python3 -c "import optuna; print(optuna.__version__)"

### Schritt 2 — Objective Function
Neue Funktion _optuna_objective(trial, strategy, asset, conn):
- trial.suggest_int / suggest_float für jeden Parameter der Strategie
- Backtest mit apply_costs=True für alle 3 WF-Fenster ausführen
- Gibt fitness_score zurück (aus _passes() Logik)
- Bei _passes()=False: return 0.0 (schlechtester Score)
- Pruner: nach Fenster 1 bereits prüfen ob pf < 1.20 → trial.report + raise TrialPruned

### Schritt 3 — Study pro (strategy, asset)
study = optuna.create_study(
    direction="maximize",
    sampler=optuna.samplers.TPESampler(seed=42),
    pruner=optuna.pruners.MedianPruner(n_startup_trials=5)
)
study.optimize(_optuna_objective, n_trials=50)

### Schritt 4 — Integration in _run_one_target()
Ersetze batch-Loop durch study.optimize() Aufruf.
Nur beste Trials (passed=True) als Candidates für BH-Filter.
n_trials=50 statt BATCH_SIZE=20 — mehr Versuche, smarter verteilt.

### Schritt 5 — Parameter-Spaces definieren
Für jede Strategie in SIGNAL_FNS: welche Parameter sind tunable?
Beispiel donchian_breakout:
  period = trial.suggest_int("period", 10, 50)
  atr_multiplier = trial.suggest_float("atr_mult", 1.0, 3.0)
Alle anderen Strategien analog.

### Schritt 6 — Persistenz (optional, Phase 2)
Optuna-Studies in SQLite speichern für Warm-Start:
storage = f"sqlite:///data/optuna_studies.db"
Damit lernt das Lab über Runs hinweg.

## Risiken
- Optuna-Studies können Hunderte MB groß werden → SQLite-Storage erst
  nach Smoke-Test aktivieren
- Parameter-Spaces müssen je Strategie manuell definiert werden —
  aufwändigster Teil der Implementierung
- Laufzeit pro Study (50 Trials × 3 Fenster × Backtest) kann
  10–30 Min dauern — kein Problem für Daemon, aber für ersten Test beachten
