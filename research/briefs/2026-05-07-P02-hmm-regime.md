# Brief: P-02 — HMM Regime-Detector (3 States)

**Datum:** 2026-05-07
**Priorität:** P3
**Adressat:** quant-researcher

## Kontext
APEX V2 nutzt aktuell EMA50-Slope + ATR als Regime-Filter.
Das ist binär und reagiert träge. Ein 3-State Gaussian HMM
erkennt Trend / Sideways / High-Volatility als separate
Zustände und ermöglicht strategie-spezifische Aktivierung.

## Ziel-Architektur

3 Hidden States:
  State TREND    → gerichtete Bewegung, moderate Volatilität
  State SIDEWAYS → enge Range, niedrige Volatilität
  State HIGH_VOL → Spike, Crash, Expansion

Strategie-Mapping:
  donchian_breakout → [TREND, HIGH_VOL]
  mean_reversion    → [SIDEWAYS]
  squeeze           → [SIDEWAYS, TREND]
  ema_pullback      → [TREND]
  alle anderen      → [TREND, SIDEWAYS, HIGH_VOL] (kein Filter)

## Features (4 Dimensionen)

  log_return_1h  = log(close / close_prev_1h)
  log_return_4h  = log(close_4h / close_4h_prev)
  atr_ratio      = ATR(14) / close
  volume_ratio   = volume / volume_rolling_20_mean

Alle Features werden vor Training und Inference
mit StandardScaler normalisiert.

## Implementierungsplan

### Schritt 1 — Installation
  pip install hmmlearn scikit-learn --break-system-packages
  Verifikation: python3 -c "from hmmlearn import hmm; print('OK')"

### Schritt 2 — Training-Script: research/train_hmm.py

Funktionen:
  load_features(asset, conn, lookback_days=180)
    Liest 1h-Candles aus DB, berechnet 4 Features,
    gibt numpy Array Shape (T, 4) zurück.
    4h-Returns: durch Resampling der 1h-Bars (jede 4. Bar).

  train_hmm(asset, conn)
    model = GaussianHMM(
        n_components=3,
        covariance_type="full",
        n_iter=200,
        n_init=10,
        random_state=42,
        tol=1e-4
    )
    Wichtig: n_init=10 für stabile Konvergenz.
    Fallback: wenn "full" nicht konvergiert → "diag".
    Raises RuntimeError wenn model.monitor_.converged=False.

  label_states(model)
    means = model.means_  (Shape 3x4)
    Feature-Index 2 = atr_ratio:
      HIGH_VOL  → State mit höchstem atr_ratio-Mean
    Feature-Index 0 = log_return_1h:
      TREND     → verbleibender State mit höchstem return-Mean
      SIDEWAYS  → verbleibender State

  save_model(asset, model, scaler, path="data/hmm_models/")
    Speichert dict mit model und scaler als pickle.
    Erstellt Verzeichnis falls nicht vorhanden.

### Schritt 3 — Inference: get_current_regime(asset, conn)
  Liest letzte 5 Tage Features.
  Lädt model + scaler aus data/hmm_models/{asset}_hmm_py312.pkl.
  Gibt Label des letzten Zustands zurück: TREND / SIDEWAYS / HIGH_VOL.

### Schritt 4 — HMMRegimeCheck in governance/checks.py
  Neuer Check nach DailyDrawdownCheck in der Check-Liste.
  evaluate(signal):
    regime = get_current_regime(signal.asset, conn)
    allowed = STRATEGY_ALLOWED_REGIMES.get(signal.strategy,
              ["TREND", "SIDEWAYS", "HIGH_VOL"])
    if regime not in allowed:
        return False, f"hmm_regime={regime} not in {allowed}"
    return True, f"hmm_regime={regime} OK"
  STRATEGY_ALLOWED_REGIMES als Konstante in config/settings.py.

### Schritt 5 — Re-Training Timer
  Script: scripts/run_hmm_retrain.py
    Iteriert alle Assets in active_deployments.
    Ruft train_hmm() + save_model() für jedes Asset auf.
    Schreibt Heartbeat: component="hmm_retrain".
  systemd: apex-hmm-retrain.service + apex-hmm-retrain.timer
    OnCalendar: Sun 05:00:00 (wöchentlich Sonntag 05:00)

## Akzeptanzkriterien
- GaussianHMM konvergiert für SOL (monitor_.converged=True)
- Transitions-Matrix Diagonale > 0.85 (Regime-Persistenz)
- State-Labels korrekt: HIGH_VOL hat höchsten atr_ratio-Mean
- get_current_regime() gibt stabiles Ergebnis letzte 24h
- HMMRegimeCheck in Governance aktiv und blockiert korrekt
- python3 -m py_compile alle geänderten Dateien OK
- parity_test: 12 PASS, 0 FAIL

## Risiken
- State-Index nie direkt verwenden — immer label_states()
- Wenig HIGH_VOL Daten (seltene Events) → covariance "diag" Fallback
- STRATEGY_ALLOWED_REGIMES initial konservativ → lieber zu restriktiv
- Pickle-Modelle nicht mit anderen Python-Versionen kompatibel →
  Versionsnummer in Dateiname einbauen: {asset}_hmm_py312.pkl
